    ans:

    add = (+)

    subtract = (-)

    multiply = (*)

    divide = (/)

    remainder or mod = (%)

    exponentiation = (**)
        Basicaly number to the power of...

        const num1 = 2;

        console.log(exponent); // 8

        const result = 10 + 5 * 2 - 8 / 4;

    Note: JS evaluates mixed expressions using operator precedence, which 
    determines the order in which operations are performed.

3) What Happens When You Try to Do Calculations with Numbers and Strings?
    ans:

    JavaScript can automatically convert one data type into another. This 
    is called type coercion. It commonly happens when you mix numbers, 
    strings, booleans, null, or undefined in arithmetic operations.

    Addition (+)
    The + operator does both adding number and string concatenation

        const result = 5 + '10';
        console.log(result); // 510
        console.log(typeof result); // string

    The order doesnt matter

        const result = '10' + 5;
        console.log(result); // 105
        console.log(typeof result); // string
    
    For subtraction (-), multiplication (*), and division (/), JavaScript tries to convert the string into a number before performing the operation.

        const subtractionResult = '10' - 5;
        console.log(subtractionResult); // 5
        console.log(typeof subtractionResult); // number

        const multiplicationResult = '10' * 2;
        console.log(multiplicationResult); // 20
        console.log(typeof multiplicationResult); // number

        const divisionResult = '20' / 2;
        console.log(divisionResult); // 10
        console.log(typeof divisionResult); // number

    If the string cannot be converted to a valid number, JavaScript returns NaN (Not a Number).
        const subtractionResult = 'abc' - 5;
        console.log(subtractionResult); // NaN
        console.log(typeof subtractionResult); // number

        const multiplicationResult = 'abc' * 2;
        console.log(multiplicationResult); // NaN
        console.log(typeof multiplicationResult); // number

        const divisionResult = 'abc' / 2;
        console.log(divisionResult); // NaN
        console.log(typeof divisionResult); // number

    Although NaN means "Not a Number," its type is still number.

    In mathematical operations:
        true becomes 1
        false becomes 0

        const result1 = true + 1;
        console.log(result1); // 2
        console.log(typeof result1); // number

        const result2 = false + 1;
        console.log(result2); // 1
        console.log(typeof result2); // number

        const result3 = 'Hello' + true;
        console.log(result3); // "Hellotrue"
        console.log(typeof result3); // string

    null and undefined in mathematical operations:
        null is treated as 0
        undefined is treated as NaN

        const result1 = null + 5;
        console.log(result1); // 5
        console.log(typeof result1); // number

        const result2 = undefined + 5;
        console.log(result2); // NaN
        console.log(typeof result2); // number
